# practice
## 1. 플레이 가이드 및 실행 방법

이 게임은 콘솔창에서 텍스트 입력만으로 진행되는 직관적인 RPG입니다. 플레이어는 게임 화면의 지시에 따라 번호를 입력하며 게임을 진행할 수 있습니다.

### 플레이 흐름
1. **직업 선택:** 게임이 시작되면 [1] 전사, [2] 마법사, [3] 궁수 중 원하는 직업의 번호를 입력하여 캐릭터를 생성합니다.
2. **메인 메뉴 조작:**  `1`을 입력하면 사냥터(전투)에 입장합니다.
   * `2`를 입력하면 현재 보유 중인 5칸의 인벤토리 상태와 아이템 설명을 실시간으로 확인할 수 있습니다.
   * `3`을 입력하면 안전하게 게임을 종료합니다.
3. **턴제 전투 진행:** 사냥터 입장 후 매 턴마다 세 가지 행동 중 하나를 선택해야 합니다.
   * `1. 일반 공격`: 적에게 기본 피해를 주며, **고유 스킬 게이지를 1칸 충전**합니다.
   * `2. 고유 스킬 사용`: 일반 공격을 통해 **게이지를 2칸 이상 채웠을 때만 활성화**되며, 직업별 고유 스킬을 사용합니다. (조건 미달 시 사용 불가)
   * `3. 아이템 사용`: 인벤토리에 남은 소모품 번호를 선택하여 즉시 체력 회복, 공격력 반영구 버프, 적 공격력 디버프 등의 효과를 발동합니다.
4. **스테이지 클리어 및 전리품:** 각 스테이지마다 적의 HP를 0으로 만들면 승리하며, 몬스터가 드롭한 전리품이 인벤토리의 빈 슬롯에 자동으로 저장됩니다. 총 3개의 스테이지를 돌파해 최종 보스 '마왕'을 물리치면 게임에서 승리합니다.

## 2. 개발자 환경 빌드 방법

별도의 외부 라이브러리 설치 없이 **C++ 표준 라이브러리**만으로 작동합니다.

1. Visual Studio를 실행하고 **[C++ 빈 프로젝트]**를 생성합니다.
2. 본 저장소의 파일들을 프로젝트 내부 폴더에 배치하고 다음과 같이 등록합니다.
   * **헤더 파일 (.h):** `Character.h`, `Item.h`
   * **소스 파일 (.cpp):** `main.cpp`, `Character.cpp`, `Item.cpp`
3. 단축키 `Ctrl + F5` 또는 `F5`를 눌러 컴파일 후 게임을 실행합니다.
---

## 3. 작품 소개

이 게임은 클래스의 모듈화, 객체 배열 관리, 동적 타겟팅을 위한 포인터 활용 등 **C++의 핵심적인 메모리 관리 구조와 객체 지향 개념을 실전 게임 루프에 녹여낸 텍스트 기반 RPG**입니다. 

플레이어는 고유한 스킬과 능력치를 지닌 3가지 직업 중 하나를 선택하여, 총 3단계로 이루어진 스테이지를 돌파하고 최종 보스인 '마왕'을 토벌해야 합니다. 실시간 인벤토리 관리와 턴제 전투를 통해 전략적인 재미를 제공합니다.

---

## 4. 주요 기능

* **직업 선택 시스템:** 전사(안정성/흡혈), 마법사(고화력/반동 패널티), 궁수(확률형 연속 공격) 등 플레이어 성향에 맞는 3종의 직업 제공
* **실시간 턴제 전투:** 플레이어의 행동(공격, 스킬, 아이템) 이후 몬스터가 동적으로 반격하는 유기적인 턴 시스템
* **스킬 게이지 시스템:** 무분별한 스킬 남발을 막기 위해, 일반 공격을 최소 2회 성공시켜야만 고유 스킬이 활성화되는 밸런스 제어
* **확장형 인벤토리 및 전리품 드롭:** 스테이지 클리어 시 몬스터가 드롭하는 전리품이 인벤토리의 빈 슬롯을 탐색하여 동적으로 추가되는 시스템

## 5. 기술적 포인트

본 프로젝트에서 안정성과 가독성, 그리고 C++ 고유의 강점을 살리기 위해 가장 신경 쓴 기술적 핵심 요소들입니다.

### ① 의존성 꼬임 방지를 위한 다중 파일 분할
클래스의 구조를 정의하는 헤더 파일과 상세 로직을 담당하는 구현 파일을 분리했습니다. 헤더의 중복 정의를 방지했습니다.

### ② 참조와 포인터의 활용을 통한 메모리 최적화
전투를 치를 때 매 턴마다 객체 전체가 복사되어 메모리가 낭비되는 것을 막기 위해, 대상의 주소값만 전달하는 **포인터 간접 참조 연산자**와 **참조자**를 적극적으로 활용했습니다. 특히 전역 함수에서 플레이어는 참조로 안전하게 제어하고, 적 몬스터는 포인터로 받아 스테이지 변동에 따라 동적으로 타겟팅이 바뀌도록 설계했습니다.

### ③ 예외 처리
* **동시 사망 연산 예외:** 마법사의 스킬 반동 데미지로 인해 플레이어와 몬스터가 동시에 사망하는 특수 상황을 고려하여, 플레이어의 생존 조건 검사를 최우선 순위로 배치해 로직이 꼬이는 것을 방지했습니다.
